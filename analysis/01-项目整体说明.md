# Claude Code 源码解读：项目整体说明

## 项目概述

Claude Code（内部代号 Tengu）是 Anthropic 开发的一款 AI 编程助手，以终端 CLI 为主要交互界面。它将 Claude 大语言模型深度集成到开发者的日常工作流中，让 AI 能够直接读写文件、执行命令、搜索代码、管理 Git 操作等。

与传统的 IDE 插件不同，Claude Code 是一个独立的终端应用程序，使用 React + Ink 构建终端 UI，通过 Bun 运行时执行，提供了一个完整的 AI 编程环境。

## 技术栈

| 层面 | 技术选型 |
|------|---------|
| 运行时 | Bun |
| 语言 | TypeScript |
| UI 框架 | React + Ink（终端 UI） |
| API 通信 | Anthropic Claude API（流式） |
| 协议支持 | MCP（Model Context Protocol） |
| 状态管理 | 自研轻量 Store（发布-订阅模式） |
| Schema 验证 | Zod |
| 构建 | Bun bundler（含 feature flag 条件编译） |

## 核心功能模块

### 1. 工具系统（Tools）

项目的核心能力通过 40+ 个工具实现，每个工具都是一个独立模块：

- **文件操作**：`FileReadTool`、`FileWriteTool`、`FileEditTool`（精确字符串替换）
- **命令执行**：`BashTool`、`PowerShellTool`（带沙箱隔离）
- **代码搜索**：`GlobTool`（文件搜索）、`GrepTool`（内容搜索）
- **Web 能力**：`WebSearchTool`、`WebFetchTool`
- **Agent 系统**：`AgentTool`（子 Agent 派生）、`SendMessageTool`（Agent 间通信）
- **任务管理**：`TaskCreateTool`、`TaskListTool`、`TaskUpdateTool` 等
- **MCP 集成**：`MCPTool`（调用外部 MCP 服务器工具）
- **LSP 集成**：`LSPTool`（语言服务器协议）
- **其他**：`TodoWriteTool`、`NotebookEditTool`、`ConfigTool`、`SkillTool` 等

### 2. 多 Agent 协作（Swarm）

支持多种 Agent 协作模式：

- **子 Agent 派生**：主 Agent 可以创建子 Agent 处理独立任务
- **Fork 模式**：子 Agent 继承父 Agent 的完整上下文
- **后台 Agent**：支持在后台运行 Agent 任务
- **Tmux/iTerm 集成**：通过终端分屏实现可视化的多 Agent 协作
- **In-Process Teammate**：进程内的轻量级 Agent 协作
- **Remote Agent**：远程环境中运行的 Agent

### 3. 权限与安全系统

多层安全防护机制：

- **工具权限控制**：每个工具调用前需要权限检查
- **沙箱执行**：命令在受限沙箱中执行，控制文件系统和网络访问
- **自动模式**：支持自动批准安全操作，危险操作仍需确认
- **分类器审批**：基于 AI 分类器的自动权限决策
- **路径验证**：文件操作的路径白名单/黑名单机制
- **Git 安全协议**：防止危险的 Git 操作

### 4. 上下文管理

智能的上下文窗口管理：

- **自动压缩（Auto Compact）**：上下文接近限制时自动压缩历史对话
- **微压缩（Micro Compact）**：对单个工具结果进行精细压缩
- **会话记忆（Session Memory）**：跨对话保持关键信息
- **CLAUDE.md 配置**：项目级别的指令和上下文注入
- **记忆提取**：自动从对话中提取有价值的信息保存

### 5. MCP 协议支持

完整的 MCP 客户端和服务器实现：

- **MCP 客户端**：连接外部 MCP 服务器，使用其提供的工具
- **MCP 服务器模式**：Claude Code 自身可作为 MCP 服务器暴露工具
- **OAuth 认证**：支持远程 MCP 服务器的 OAuth 认证流程
- **批量连接**：并行连接多个 MCP 服务器

### 6. 钩子系统（Hooks）

可扩展的事件钩子机制：

- **Pre/Post Tool Use**：工具执行前后的拦截
- **Session Start/End**：会话生命周期钩子
- **Permission Request**：权限请求拦截
- **Compact**：压缩前后的处理
- **File Changed**：文件变更通知
- **自定义钩子**：用户可通过配置文件定义钩子

### 7. 技能系统（Skills）

内置和可扩展的技能：

- **内置技能**：`/commit`、`/commit-push-pr`、`/loop`、`/batch`、`/debug` 等
- **自定义技能**：用户可定义自己的技能
- **技能发现**：根据上下文自动推荐相关技能

### 8. 命令系统

80+ 个斜杠命令，覆盖各种操作：

- **会话管理**：`/clear`、`/compact`、`/resume`、`/session`
- **配置管理**：`/config`、`/model`、`/theme`、`/permissions`
- **开发工具**：`/diff`、`/review`、`/doctor`、`/stats`
- **集成功能**：`/mcp`、`/ide`、`/chrome`、`/desktop`
- **团队协作**：`/agents`、`/tasks`、`/skills`

### 9. IDE 集成

支持与多种 IDE 的集成：

- VS Code 扩展集成
- JetBrains IDE 集成
- Chrome 浏览器扩展
- 桌面应用模式

### 10. 其他功能

- **语音输入**：支持语音转文字输入（Sox/arecord）
- **成本追踪**：实时追踪 API 调用成本，支持按模型分类统计
- **会话恢复**：支持断点续传和会话恢复（`/resume`）
- **会话分支**：从某个时间点创建对话分支（`/branch`）
- **自动更新**：内置更新检查和升级机制
- **Vim 模式**：完整的 Vim 键绑定支持（motions、operators、text objects）
- **主题系统**：可自定义终端主题（亮色/暗色）
- **导出功能**：对话导出为多种格式
- **Teleport**：跨机器的会话迁移（Git bundle + 远程环境）
- **Auto Dream**：后台自动执行的记忆整理任务
- **Advisor Tool**：服务端工具，让更强的模型审查主模型的工作
- **Undercover 模式**：在公开仓库工作时隐藏 AI 身份，防止泄露内部信息
- **Fast Mode**：快速推理模式（Opus 4.6 支持，6x 价格换取更快速度）
- **Effort 级别**：low/medium/high/max 四档推理深度控制
- **Plan 模式**：只允许只读工具的规划模式，写入需要确认
- **Token Budget**：用户可指定 token 预算（如 `+500k`），AI 持续工作直到接近目标
- **Cron 调度**：支持 cron 表达式的定时任务
- **Deep Link**：`claude://` 协议支持，可从浏览器直接打开 Claude Code

### 11. 记忆系统（Memory）

多层记忆架构：

- **CLAUDE.md 指令**：项目级、用户级、目录级的配置指令，支持 frontmatter 控制注入时机
- **会话记忆（Session Memory）**：自动从对话中提取关键信息，跨轮次保持
- **自动记忆提取（Extract Memories）**：后台自动将有价值的信息写入记忆文件
- **团队记忆同步（Team Memory Sync）**：团队间共享记忆，含秘密扫描防护
- **Memory Directory（memdir）**：结构化的记忆文件目录，支持按类型分类

### 12. Bridge 远程桥接

支持远程开发场景：

- **REPL Bridge**：本地终端与远程环境的桥接
- **Session Runner**：远程会话管理
- **Capacity Wake**：远程容量唤醒
- **JWT 认证**：安全的远程连接认证
- **Headless 模式**：无头运行模式，支持 CI/CD 集成

### 13. Companion 系统（Buddy）

一个有趣的虚拟宠物系统：

- 基于用户 ID 的确定性随机生成
- 包含物种、眼睛、帽子、稀有度等属性
- RPG 风格的属性系统（stats）
- 终端精灵动画渲染

### 14. 自定义 Ink 渲染引擎

项目内置了一个完整的终端 UI 渲染引擎（96 个文件）：

- 自定义 Yoga 布局引擎（纯 TS 实现，替代 C++ 绑定）
- 完整的组件系统（Box、Text、Button、Link 等）
- 事件系统（点击、输入、焦点）
- 动画支持
- BiDi 文本支持
- 主题系统

### 15. 数据迁移系统

支持配置和模型的平滑迁移：

- 模型迁移（Fennec → Opus → Opus 1M → Sonnet 4.5 → Sonnet 4.6）
- 设置迁移（权限、自动更新、MCP 配置等）
- 向后兼容保证

## 项目规模

- 源码文件数：800+ 个 TypeScript/TSX 文件
- 自定义 Ink 渲染引擎：96 个文件
- 工具数量：40+ 个内置工具
- 命令数量：80+ 个斜杠命令
- 服务模块：20+ 个服务
- 工具函数：300+ 个工具函数
- 钩子系统：5000+ 行核心代码，50+ 种事件类型
- MCP 客户端：3200+ 行，支持完整的 MCP 协议

## 项目定位

Claude Code 不仅仅是一个 AI 聊天工具，它是一个完整的 AI 编程平台，具备：

1. **深度系统集成**：直接操作文件系统、执行命令、管理 Git
2. **多 Agent 架构**：支持复杂任务的分解和并行处理
3. **安全优先**：多层安全机制保护用户环境
4. **可扩展性**：通过 MCP、钩子、技能等机制支持扩展
5. **开发者体验**：终端原生、快捷键支持、Vim 模式等
