# Claude Code 源码解读：项目架构文档

## 技术栈概览

| 层面 | 技术 | 说明 |
|------|------|------|
| 运行时 | Bun | 高性能 JS/TS 运行时，支持 `bun:bundle` feature flag |
| 语言 | TypeScript | 全量 TS，Zod 做运行时 schema 校验 |
| 终端 UI | React + Ink | 基于 React 的终端渲染框架（自定义 96 文件） |
| API | Anthropic SDK | 流式 SSE 通信 |
| 协议 | MCP | Model Context Protocol 客户端/服务器 |
| 状态管理 | 自研 Store | 轻量发布-订阅模式（30 行实现） |

## 核心对话循环流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant REPL as REPL 循环
    participant QE as QueryEngine
    participant API as Claude API
    participant TO as 工具编排器
    participant Tool as 工具
    participant Perm as 权限系统

    User->>REPL: 输入消息
    REPL->>QE: submitMessage()
    
    loop 对话循环 (queryLoop)
        QE->>QE: 构建系统提示词
        QE->>QE: 注入上下文附件
        QE->>API: 流式请求 (queryModel)
        
        loop 流式响应
            API-->>QE: stream event (text/tool_use)
            QE-->>REPL: yield StreamEvent
        end
        
        alt 包含工具调用
            QE->>TO: runTools(toolUseBlocks)
            
            loop 工具批次处理
                TO->>TO: partitionToolCalls()
                
                alt 并发安全的工具
                    TO->>Tool: 并行执行多个工具
                else 非并发安全
                    TO->>Tool: 串行执行
                end
                
                Tool->>Perm: checkPermissions()
                Perm-->>Tool: 允许/拒绝
                
                alt 需要用户确认
                    Tool-->>User: 权限请求对话框
                    User-->>Tool: 批准/拒绝
                end
                
                Tool-->>TO: 工具结果
            end
            
            TO-->>QE: 工具结果消息
            Note over QE: 继续下一轮循环
        else 无工具调用 (end_turn)
            QE-->>REPL: 返回最终响应
        end
        
        opt 上下文过大
            QE->>QE: autoCompact / microCompact
        end
    end
    
    REPL-->>User: 显示响应
```

## 工具执行流程

```mermaid
flowchart TD
    A[收到工具调用请求] --> B{工具是否启用?}
    B -->|否| C[返回错误]
    B -->|是| D[验证输入 validateInput]
    D -->|无效| C
    D -->|有效| E[执行 Pre-Tool Hooks]
    E --> F{Hook 是否阻止?}
    F -->|是| G[返回阻止消息]
    F -->|否| H[检查权限 checkPermissions]
    H --> I{权限结果}
    I -->|允许| J[执行工具 tool.call]
    I -->|需确认| K[显示权限对话框]
    I -->|拒绝| L[返回拒绝消息]
    K --> M{用户决定}
    M -->|批准| J
    M -->|拒绝| L
    J --> N[执行 Post-Tool Hooks]
    N --> O[返回工具结果]
```

## API 调用流程

```mermaid
sequenceDiagram
    participant QL as queryLoop
    participant QM as queryModel
    participant WR as withRetry
    participant API as Anthropic API

    QL->>QM: 调用 queryModel
    QM->>QM: 构建工具 Schema
    QM->>QM: 消息规范化 (normalizeMessagesForAPI)
    QM->>QM: 添加缓存断点 (addCacheBreakpoints)
    QM->>QM: 配置 thinking/effort/taskBudget
    QM->>WR: withRetry 包装
    WR->>API: messages.create(stream: true)
    
    loop 流式事件
        API-->>QM: message_start (TTFT)
        API-->>QM: content_block_start
        API-->>QM: content_block_delta (text/tool_use/thinking)
        API-->>QM: content_block_stop
        API-->>QM: message_delta (usage/stop_reason)
        API-->>QM: message_stop
    end
    
    QM->>QM: 流式空闲看门狗 (watchdog)
    QM->>QM: 流式停顿检测 (stall detection)
    
    alt 流式失败
        QM->>QM: 降级为非流式请求
    end
    
    QM-->>QL: yield AssistantMessage / StreamEvent
```
