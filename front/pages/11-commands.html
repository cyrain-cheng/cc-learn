⌨️ 命令与技能

80+ 斜杠命令 · 三种命令类型 · 技能系统 · 插件系统

整体实现思路

Claude Code 提供了 80+ 个斜杠命令 + 可扩展技能系统 + 插件系统，构成用户与 AI 交互的命令层。设计理念："轻量入口、统一注册、多源合并"。

命令类型

类型执行方式示例
prompt构建消息发送给 AI，AI 处理后返回结果/commit、/review、/commit-push-pr
local直接执行本地逻辑，不经过 AI/clear、/compact、/voice
local-jsx渲染 React 组件（设置面板、选择器等）/config、/resume、/model
 
type Command = {
  type: 'prompt' | 'local' | 'local-jsx'
  name: string
  description: string
  allowedTools?: string[]       // prompt 类型：限制 AI 可用的工具
  source: 'builtin' | 'bundled' | 'plugin' | 'mcp'
  getPromptForCommand?(args, context): ContentBlockParam[]
  load?(): Promise<{ call: CommandCall }>
}

关键命令

命令类型说明
/commitprompt分析 git diff 生成 commit message，限制工具为 git add/commit
/commit-push-prprompt在 /commit 基础上增加 push 和创建 PR
/clearlocal清除对话消息，触发 SessionEnd 钩子，重置上下文
/compactlocal手动触发上下文压缩
/resumelocal-jsx渲染会话选择器，恢复历史会话
/branchlocal-jsx从当前对话创建分支
/configlocal-jsx渲染配置面板
/modellocal-jsx模型选择器

技能 vs 命令：技能本质上是命令，但额外支持自动发现和推荐（AI 可以主动建议使用某个技能）。

用户输入处理

1
以 / 开头 → 斜杠命令解析
findCommand 查找匹配命令，未找到则建议相似命令
2
以 ! 开头 → Bash 命令
直接执行 shell 命令
3
普通文本 → 文本提示
发送给 AI 处理

学习要点

命令是轻量入口
实际逻辑委托给服务层，命令本身只负责参数解析和路由
prompt 类型最巧妙
通过构建提示词让 AI 执行复杂任务，同时限制可用工具确保安全
四源合并统一命令池
内置 + 技能 + MCP + 插件在运行时合并，用户无感知
插件支持多种安装方式
Git、NPM、本地、Marketplace，覆盖不同分发场景
