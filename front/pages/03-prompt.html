📝 提示词系统

提示词即代码 · 多层动态组合 · 缓存优先 · 分层组合

整体实现思路

提示词系统是整个项目最精妙的部分之一。它不是一个静态字符串，而是一个多层动态组合的工程体系，由三个层面构成：系统提示词（告诉 AI 它是谁）、工具提示词（告诉 AI 每个工具怎么用）、上下文注入（把运行时信息动态注入对话）。

💻
提示词即代码
通过函数动态生成，支持条件编译（feature() 门控）、运行时注入、版本差异
💾
缓存优先
静态/动态有明确分界线，静态部分可跨用户全局缓存，直接影响 API 成本
🧩
分层组合
支持覆盖、替换、追加三种模式，不同场景有不同的组合策略

8 种工具提示词设计模式

模式一：工具偏好引导（BashTool）

当系统有多个工具时，明确告诉 AI 什么场景用什么工具，用 NOT 强调不要用什么。

IMPORTANT: Avoid using this tool to run cat, head, tail...
Instead, use the appropriate dedicated tool:
- File search: Use GlobTool (NOT find or ls)
- Content search: Use GrepTool (NOT grep or rg)
- Read files: Use Read (NOT cat/head/tail)
模式二：NEVER 红线（Git 安全协议）

对于危险操作，使用 NEVER + CRITICAL 强调词，并给出明确的例外条件。

Git Safety Protocol:
- NEVER update the git config
- NEVER run destructive git commands (push --force, reset --hard...)
- CRITICAL: Always create NEW commits rather than amending
模式三：结构化操作指南（Git 提交）

对于复杂多步骤操作，使用编号步骤，并标注哪些可以并行、哪些必须串行。

1. Run the following bash commands in parallel:
   - Run a git status command...
   - Run a git diff command...
2. Analyze all staged changes and draft a commit message
3. Run the following commands in parallel:
   - Add relevant untracked files
   - Create the commit
模式四：角色比喻（AgentTool）

用比喻帮助 AI 理解子 Agent 的上下文限制。

Brief the agent like a smart colleague who just walked
into the room — it hasn't seen this conversation,
doesn't know what you've tried, doesn't understand
why this task matters.
模式五：反模式警告（AgentTool）

不仅告诉 AI 该做什么，还明确告诉它不该做什么，并解释为什么。

**Never delegate understanding.** Don't write
"based on your findings, fix the bug" — those phrases
push synthesis onto the agent instead of doing it yourself.
模式六：命令式短语（Fork 指导）

用简短有力的命令式短语建立行为规则。

**Don't peek.** The tool result includes an output_file path —
do not Read or tail it unless the user explicitly asks.

**Don't race.** After launching, you know nothing about
what the fork found. Never fabricate or predict fork results.
模式七：消除犹豫（FileReadTool）

明确告诉 AI 可以大胆尝试，错误会被优雅处理。

Assume this tool is able to read all files on the machine.
If the User provides a path to a file assume that path is valid.
It is okay to read a file that does not exist; an error will be returned.
模式八：强制输出格式（WebSearchTool）

对于必须遵守的规则，使用 CRITICAL REQUIREMENT + MUST + MANDATORY 三重强调。

CRITICAL REQUIREMENT - You MUST follow this:
- After answering, you MUST include a "Sources:" section
- This is MANDATORY - never skip including sources

Prompt Cache 5 大优化策略

1
静态/动态分界线
SYSTEM_PROMPT_DYNAMIC_BOUNDARY — 静态部分可跨用户全局缓存
2
Beta Header 锁存
一旦发送某个 header，整个会话保持不变，防止缓存 key 变化
3
路径规范化
$TMPDIR 替代 /tmp/claude-1001/，使提示词跨用户一致
4
工具 Schema 去重
沙箱配置去重，节省 150-200 tokens/请求
5
动态部分注册表
systemPromptSection() 缓存，避免每轮重新计算
