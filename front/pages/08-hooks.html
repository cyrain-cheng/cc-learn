🪝 钩子系统

50+ 事件类型 · 模式匹配 · 结构化输出 · 三种执行方式

整体实现思路

钩子系统是 Claude Code 的事件驱动可扩展性框架，支持 50+ 种事件类型，允许外部脚本和内部回调在关键节点拦截、修改、扩展 AI 的行为。设计理念："不修改核心代码，通过事件订阅实现定制化"。

50+ 事件类型

事件类型触发时机典型用途
PreToolUse工具执行前拦截危险操作、修改工具输入
PostToolUse工具执行后审计日志、结果后处理
PostToolUseFailure工具执行失败后错误监控、自动恢复
PermissionRequest权限请求时自定义审批逻辑
PermissionDenied权限被拒绝时拒绝统计、告警
SessionStart会话开始初始化、环境检查
SessionEnd会话结束清理、报告生成
StopAI 停止响应后处理、通知
UserPromptSubmit用户提交消息输入预处理、过滤
SubagentStart子 Agent 启动Agent 监控
PreCompact / PostCompact压缩前/后压缩策略定制
FileChanged文件变更自动格式化、lint
CwdChanged工作目录变更环境切换
Notification后台任务通知桌面通知
WorktreeCreateWorktree 创建Agent 隔离监控

执行流程

系统事件触发
→
收集所有钩子源
内置 + 配置 + 插件
→
getMatchingHooks
模式匹配 + 条件判断
→
hookDedupKey 去重
→
依次执行钩子
→
验证输出 Schema

匹配逻辑

1
事件类型匹配
钩子注册的事件类型与当前事件一致
2
Matcher 匹配
工具名 Glob 匹配 → Regex 匹配（用于 PreToolUse/PostToolUse）
3
条件表达式求值
可选的条件表达式，为 true 时才匹配
4
去重
hookDedupKey 确保相同钩子对同一事件只执行一次

输出 Schema

通用字段
continue: boolean       // 是否继续执行
suppressOutput: boolean // 隐藏钩子输出
stopReason: string      // 停止原因
decision: 'approve' | 'block'  // 权限决策
reason: string          // 决策原因
systemMessage: string   // 显示给用户的警告
PreToolUse 特有输出
hookSpecificOutput: {
  hookEventName: 'PreToolUse',
  permissionDecision: ...,    // 覆盖权限决策
  updatedInput: {...},        // 修改工具输入参数
  additionalContext: '...',   // 注入额外上下文
}
SessionStart 特有输出
hookSpecificOutput: {
  hookEventName: 'SessionStart',
  initialUserMessage: '...',  // 设置初始用户消息
  watchPaths: ['...'],        // 注册文件监听路径
}

三种执行方式

方式实现适用场景
命令行脚本execCommandHook 执行 shell 命令本地脚本、CI/CD 集成
HTTP Webhook发送 HTTP POST 请求远程服务、微服务架构
内部回调直接调用 JS 函数插件系统、内置扩展

命令行脚本通过环境变量传递事件上下文，stdout 输出作为 JSON 解析，支持超时控制。

配置示例

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "pattern": "rm -rf *",
        "type": "command",
        "command": "/path/to/safety-check.sh"
      }
    ],
    "SessionStart": [
      {
        "type": "command",
        "command": "echo '{\"hookSpecificOutput\":{...}}'"
      }
    ]
  }
}

学习要点

钩子是最大的可扩展性入口
50+ 事件类型覆盖 AI 工作流的所有关键节点
PreToolUse 最强大
可以修改权限决策和工具输入，实现深度定制
结构化输出确保可控性
Zod Schema 验证返回值，防止格式错误导致系统异常
