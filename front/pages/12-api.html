📡 API通信层

流式优先 · 多层容错 · 成本可控 · 灵活配置

整体实现思路

API 通信层是 Claude Code 与 Claude API 之间的桥梁，负责流式交互、错误恢复、成本控制、缓存优化的全链路管理。核心文件 claude.ts 约 3400 行。

流式事件处理

1
message_start
记录 TTFT（首 Token 时间）
2
content_block_start
开始新的内容块（text / tool_use / thinking）
3
content_block_delta
增量内容到达，yield 给 UI 实时渲染
4
content_block_stop
内容块结束
5
message_delta → message_stop
更新 usage + stop_reason，流式结束

推理配置

Thinking 配置
adaptive — AI 自行决定是否展示思考过程
enabled — 强制开启，指定预算 Token 数
disabled — 禁用思考过程
Effort 级别
low — 快速简短回答
medium — 默认平衡
high — 深度思考
max — 最大推理深度
Fast Mode
更快推理路径，6 倍价格换取更低延迟。被 API 拒绝后进入冷却期，自动回退到正常模式。
Advisor Tool
服务端工具，让另一个 AI 模型审查主模型的工作。主模型调用 advisor → 服务端转发 → 返回审查建议。

流式健康监控

监控层阈值行为
空闲看门狗90 秒无数据中断连接，触发重试
停顿检测30 秒无数据显示告警，继续等待
非流式降级流式连续失败切换为非流式 API 调用

重试策略

1
网络错误
指数退避重试
2
速率限制（429）
等待 Retry-After 后重试
3
服务器错误（5xx）
重试，可选降级到备用模型
4
模型不可用
自动切换到 fallback 模型

Prompt Cache 优化

策略说明
Beta Header 锁存一旦发送，整个会话保持不变
缓存断点检测checkResponseForCacheBreak() — 监控 cache_read vs cache_creation
全局缓存作用域prompt_caching_scope beta — 跨用户共享缓存
Cached MicroCompactcache_edits — 编辑缓存中的内容而非重建

sideQuery 辅助调用

非主循环的 API 调用封装，用于权限解释、会话搜索、模型验证、AI 分类器等辅助功能。使用更轻量的模型降低成本。

await sideQuery({
  querySource: 'permission_explainer',
  model: 'haiku',
  system: SYSTEM_PROMPT,
  messages: [...],
  tools: [...],
  max_tokens: 1024,
})

学习要点

三层健康监控
看门狗、停顿检测、非流式降级，确保连接可靠性
Prompt Cache 直接影响成本
Beta Header 锁存和缓存断点检测是关键优化手段
重试支持模型降级
主模型不可用时自动切换到 fallback 模型
三维控制推理行为
Thinking 控制思考可见性，Effort 控制推理深度，Fast Mode 控制推理速度
