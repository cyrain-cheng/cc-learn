⚙️ 核心引擎

queryLoop 对话循环 · Tool 基类 · 工具编排 · 流式执行

整体实现思路

核心引擎是 Claude Code 的心脏，实现了"用户输入 → AI 推理 → 工具执行 → 结果反馈"的完整闭环。核心设计思想是 AsyncGenerator 驱动的流式循环：整个对话过程是一个 while(true) 循环，每一轮迭代包含 API 调用、工具执行、上下文压缩三个阶段。

🔄
流式处理
API 响应一边到达一边渲染，工具一边执行一边返回结果
⏹️
可中断
通过 AbortController 随时中断当前循环
🧩
可组合
通过 yield* 将子生成器的结果透传给父生成器
💾
内存友好
不需要缓存完整的中间结果

queryLoop 对话循环

queryLoop 是整个系统最核心的函数，位于 src/query.ts。它是一个 async function*，在 while(true) 循环中执行以下步骤：

技能预取
→
工具结果预算
→
Snip 压缩
→
MicroCompact
→
Context Collapse
→
AutoCompact
→
调用 Claude API（流式）
→
流式事件循环
→
工具执行
→
获取附件
→
继续下一轮

工具编排 — 并发控制

toolOrchestration.ts 实现了智能的工具并发控制。分区算法将工具调用分为并发安全批次和非并发安全批次：

✅ 并发安全（并行执行）
Read, Glob, Grep, WebFetch — 只读工具，最大并发 10
⚠️ 非并发安全（串行执行）
Write, Edit, Bash — 有副作用的工具，逐个执行

连续的并发安全工具合并为一个批次并行执行，遇到非并发安全工具则切换为串行。上下文修改器（contextModifier）在批次完成后统一应用。

StreamingToolExecutor — 流式工具执行

在 API 响应还在流式到达时就开始执行工具，不等完整响应：

1
收到 tool_use block
addTool() 入队，检查是否可以立即执行
2
并发安全 → 立即执行
与其他并发安全工具并行运行
3
非并发安全 → 等待
等待前面所有工具完成后再执行
4
API 流结束
getRemainingResults() 等待所有剩余工具完成

错误恢复机制

max_output_tokens — 输出截断恢复（最多 3 次）

当 AI 响应被截断时，自动继续下一轮循环让 AI 从断点继续。最多恢复 3 次，超过则停止。

prompt_too_long — 反应式压缩

API 返回 413 时触发 Reactive Compact，压缩上下文后重试。

流式失败 — 非流式降级

流式请求失败时自动降级为非流式请求。包含空闲看门狗（90s）和停顿检测（30s）。
