🤖 多Agent系统

Fork / Subagent / Teammate · AsyncLocalStorage 隔离 · 邮箱通信 · Worktree

整体实现思路

多 Agent 系统体现了从"单 Agent 独立工作"到"多 Agent 协作完成复杂任务"的演进。系统提供三种核心模式，覆盖从简单子任务到复杂并行开发的全部场景。

AsyncLocalStorage 隔离

解决同进程内多个 Agent 并发运行时的上下文隔离问题，利用 Node.js 原生能力：

const teammateContextStorage = new AsyncLocalStorage<TeammateContext>()

// 每个 Agent 在独立的 ALS 上下文中运行
function runWithTeammateContext<T>(context: TeammateContext, fn: () => T): T {
  return teammateContextStorage.run(context, fn)
}

// 任何代码路径都可以获取当前 Agent 的身份
function getTeammateContext(): TeammateContext | undefined {
  return teammateContextStorage.getStore()
}

无需显式传递上下文，异步调用链中自动继承，天然支持并发隔离。

邮箱通信机制

Agent 之间通过文件系统实现的消息队列进行通信。存储路径：~/.claude/teams/{team}/inbox/{agent}.jsonl

消息类型

IdleNotification
空闲通知
PermissionRequest/Response
权限请求与响应
ShutdownRequest
关闭请求
TaskAssignment
任务分配
PlanApprovalRequest
计划审批
DirectMessage
直接消息

基于文件系统，无需额外 IPC 机制；JSONL 格式支持增量追加；天然持久化，进程重启不丢失消息。

Agent 提示词设计

Fork 指导

告诉 Fork Agent 它继承了父上下文，应该直接执行任务而非重新收集信息。

Don't peek

禁止 Agent 读取其他 Agent 的工作目录，防止并发冲突。

Don't race

禁止 Agent 与其他 Agent 竞争同一资源（如同一文件）。

学习要点

Fork 共享缓存降低成本
适合子任务；Subagent 独立上下文，适合需要专用工具的独立任务
AsyncLocalStorage 是关键
利用 Node.js 原生能力，无需显式传递上下文，异步链自动继承
文件系统邮箱最简单可靠
JSONL 增量追加，天然持久化，无需额外 IPC 机制
