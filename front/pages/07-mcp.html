🔌 MCP协议

客户端 + 服务器双端 · 三种传输方式 · OAuth 认证 · 透明集成

整体实现思路

MCP（Model Context Protocol）是标准化的 AI 工具扩展协议。Claude Code 同时实现了客户端和服务器两端。作为客户端连接外部 MCP 服务器获取额外工具能力；作为服务器将自身 40+ 内置工具暴露给其他 MCP 客户端。

服务器模式

Claude Code 自身可作为 MCP 服务器（entrypoints/mcp.ts），暴露内置工具给外部客户端：

async function startMCPServer(cwd, debug, verbose) {
  const server = new Server({ name: 'claude/tengu', version: MACRO.VERSION })

  // 暴露所有内置工具
  server.setRequestHandler(ListToolsRequestSchema, async () => {
    const tools = getTools(emptyPermissionContext)
    return { tools: tools.map(tool => ({
      description: await tool.prompt({...}),
      inputSchema: zodToJsonSchema(tool.inputSchema),
    }))}
  })

  // 处理工具调用
  server.setRequestHandler(CallToolRequestSchema, async ({ params }) => {
    const tool = findToolByName(tools, params.name)
    const result = await tool.call(params.arguments, context)
    return { content: [{ type: 'text', text: JSON.stringify(result) }] }
  })
}

应用场景：其他 AI 工具或编辑器可以通过 MCP 协议调用 Claude Code 的文件操作、搜索、Git 等能力。

连接批量管理

类型并发数说明
本地服务器batch = 5Stdio 传输，资源开销小
远程服务器batch = 3SSE/WebSocket，需要网络连接
 
// 并行连接，但限制并发数
async function processBatched<T>(items, batchSize, fn) {
  for (let i = 0; i < items.length; i += batchSize) {
    await Promise.all(items.slice(i, i + batchSize).map(fn))
  }
}

Chrome 集成

通过 Chrome Native Messaging Host（src/utils/claudeInChrome/ 7 文件）将 MCP 能力扩展到浏览器。Chrome 扩展通过 Native Messaging 与 Claude Code 通信，Claude Code 作为 MCP 服务器处理来自浏览器的请求。

学习要点

MCP 是双向的
同时是客户端（使用外部工具）和服务器（暴露内置工具）
MCPTool 包装实现透明集成
外部工具在 AI 视角与内置工具完全一致
批量并行连接控制并发
本地 batch=5，远程 batch=3，避免资源耗尽
