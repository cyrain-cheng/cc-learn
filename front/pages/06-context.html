🧠 上下文管理

5 种压缩策略 · Token 计算 · 记忆系统 · Auto Dream

整体实现思路

LLM 的上下文窗口是有限的稀缺资源。Claude Code 通过多种压缩策略 + 持久化记忆系统智能管理上下文，确保在有限窗口内保留最有价值的信息。核心理念："分层压缩、按需触发、记忆持久化"。

5 种压缩策略（轻 → 重）

① Snip 压缩
最轻量：直接删除旧工具结果，替换为占位符。零额外成本，不调用 AI。
② MicroCompact
局部压缩：针对单个过大的工具结果，使用 Haiku 模型压缩输出内容。
③ Context Collapse
折叠旧对话段：保留每轮关键信息（意图、决策、工具调用），删除详细中间过程。
④ AutoCompact
最核心：调用 Claude 生成对话摘要替换旧消息。阈值 = 窗口 × 0.8。
⑤ Reactive Compact
紧急压缩：API 返回 prompt_too_long (413) 时触发激进 AutoCompact。

Token 计算逻辑

tokens.ts 中的 tokenCountWithEstimation 是所有压缩策略的基础：

1
找到最后一个有 usage 的 assistant 消息
API 返回的实际 Token 数
2
处理并行工具调用的消息分裂
同一 message.id 可能对应多个交错的 tool_result
3
回退到第一个分裂消息
确保所有交错的 tool_result 都被估算
4
最终结果
= usage.input_tokens + cache_tokens + output_tokens + 新消息估算

AutoCompact 触发条件

function getAutoCompactThreshold(model: string): number {
  const contextWindow = getEffectiveContextWindowSize(model)
  return Math.floor(contextWindow * 0.8)  // 阈值 = 窗口 × 0.8
}

执行流程：Token 超过阈值 → 选择压缩范围 → 调用 Claude 生成摘要 → 插入压缩边界标记 → 删除旧消息 → 恢复关键附件（CLAUDE.md、系统上下文）。

记忆系统

SessionMemory — 会话记忆

自动从对话中提取关键信息，持久化到 ~/.claude/memory/ 目录。每个记忆是独立的 .md 文件，带 frontmatter 元数据。

提取时机：AutoCompact 执行后、会话结束时、用户显式触发（/remember 命令）。

extractMemories — 记忆提取

通过后台 Fork Agent 执行，安全限制：只能读写 ~/.claude/memory/ 目录，不能修改项目代码。

// 提取提示词核心策略：
// 第 1 轮：并行读取所有可能需要更新的文件
// 第 2 轮：并行写入/编辑所有变更
// 轮次预算有限，高效策略优先

Auto Dream — 自动整理

后台自动整理和优化记忆，类似人类睡眠时的记忆整合：

检查时间间隔
→
扫描最近会话
→
获取文件锁
→
Fork Agent 整理
合并/更新/删除
→
只读 Bash 确保安全

压缩决策流程

Token 计算
→
超过阈值?
→
① Snip
→
② MicroCompact
→
③ Collapse
→
④ AutoCompact

如果 API 返回 prompt_too_long，触发 ⑤ Reactive Compact（紧急激进压缩），最多尝试一次。

学习要点

5 种策略互补而非互斥
从轻量到重量依次触发，每种解决不同粒度的问题
Token 计算处理消息分裂
并行工具调用导致的交错消息结构是计算难点
记忆提取是受限 Fork Agent
只能读写记忆目录，不能修改项目代码
Auto Dream 类似人类记忆整合
后台定期清理冗余和过时记忆
