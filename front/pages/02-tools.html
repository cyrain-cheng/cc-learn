🔧 工具系统

工具即能力单元 · 自包含模块 · 声明式并发 · 动态组合

整体实现思路

工具系统是 Claude Code 的能力层——AI 能做什么，完全由它拥有的工具决定。每个工具是一个自包含模块，包含输入 Schema（Zod）、执行逻辑（call）、使用说明（prompt）、权限检查（checkPermissions）、UI 渲染（render*）五个维度。

📦
自包含
工具的提示词和实现放在同一目录，修改工具不需要改动核心代码
⚡
声明式并发
每个工具声明自己是否并发安全，编排器据此决定并行/串行
🔗
动态组合
内置工具 + MCP 工具 + 插件工具在运行时合并为统一工具池

工具清单

分类工具说明
文件Read / Write / Edit文件读取（支持图片/PDF/Notebook）、写入、精确字符串替换

搜索Glob / Grep / ToolSearch文件名搜索、内容搜索、工具搜索
WebWebSearch / WebFetch网页搜索、网页抓取并转 Markdown
AgentAgent / SendMessage子 Agent 派生、队友消息通信
MCPMCPTool外部 MCP 服务器工具的统一包装
任务Task* / Todo任务管理、待办事项

单工具目录结构

src/tools/SomeTool/
├── SomeTool.ts/tsx    # 主文件 buildTool({...})
├── prompt.ts          # 提示词（告诉 AI 怎么用）
├── UI.tsx             # 渲染组件
├── constants.ts       # 常量
├── utils.ts           # 辅助函数
└── *Security.ts       # 安全检查

BashTool 架构（18 文件，最复杂）

BashTool 是整个工具系统中最复杂的工具，涵盖安全检查、权限逻辑、命令语义分析、沙箱判断等多个子模块。

文件职责
BashTool.tsx主入口，串联所有子模块
prompt.ts400 行提示词，含 Git 安全协议
bashSecurity.ts安全检查总入口
bashPermissions.ts权限逻辑
commandSemantics.ts命令语义分析
modeValidation.ts模式验证
pathValidation.ts路径验证
readOnlyValidation.ts只读验证
sedValidation.tssed 命令验证
shouldUseSandbox.ts沙箱判断
destructiveCommandWarning.ts破坏性命令警告

工具注册流程

getAllBaseTools()
40+ 内置工具
→
feature flag
条件编译
→
filterToolsByDenyRules
权限过滤
→
assembleToolPool
合并 MCP 工具
→
getMergedTools
最终工具池

排序规则：内置工具（按名称）在前，MCP 工具（按名称）在后 → 保证 prompt cache 稳定性。

学习要点

工具是自包含的
新增工具只需创建目录，不改核心代码
提示词是工具的 API 文档
告诉 AI 什么时候用、怎么用、不要怎么用
readEditContext 性能优化
大文件只读匹配位置周围的上下文，不加载整个文件
工具排序影响 prompt cache
内置在前 MCP 在后，按名称排序保证稳定
